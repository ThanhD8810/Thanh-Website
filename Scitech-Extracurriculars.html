📘 Grade 9 — Mr Knowles' Grade 9 Coding Club

This was my first experience with programming outside of the classroom. We used Unity and C# to build both 2D and 3D games, and my biggest accomplishment that year was developing a 3D car driving simulator. It was a fully navigable terrain with realistic road textures and vehicle physics. I worked hard to understand how to script acceleration, camera angles, and turning mechanics. The learning curve was steep, especially in writing scripts that made the car behave properly and react to input smoothly, but I loved it. Creating the simulator taught me the value of trial and error, how to use object hierarchies in Unity, and how to debug in real time. Watching a virtual car drive across a map that I built from scratch made me realize how powerful code can be. This experience gave me confidence and solidified my interest in computer-aided engineering and software development.

📙 Grade 10 — Coding Club (Year 2)

This year we advanced into object-oriented programming, web development, and Python-based projects. I helped newer members with syntax and logic flow, and for my own learning, I dove into building websites from scratch. One of my proudest achievements was creating my first Tesla clone website using HTML, CSS, and JavaScript. The site mimicked Tesla's sleek UI design and had smooth scrolling sections, animated car images, and responsive layouts. I learned how to structure a webpage, link stylesheets and scripts, and use modern development tools like GitHub for version control. It was the first time I felt like I had the tools to bring professional-looking ideas to life. 📗 Grade 12 — STEM Student Association

In just 30 minutes, my friends and I built a popsicle stick bridge during a timed STEM Student Association competition. Despite the rushed design and limited materials, our bridge ended up being the strongest of all entries and—according to the very unbiased judges—also the best looking. What I enjoyed the most was the spontaneous teamwork and laughter as we scrambled to put the structure together. It was one of those moments where creativity and problem-solving came together under pressure. The biggest challenge was working against the clock with fragile materials. It forced us to think fast, compromise on aesthetics and structure, and trust each other’s instincts. I chose to participate because it combined competition with hands-on engineering, which is the heart of what I love about STEM. The biggest takeaway was that even a “sketchy” idea can succeed with the right execution and collaboration. It reminded me that sometimes the best solutions aren’t the prettiest on paper—they’re the ones that simply work.

📗 Grade 12 — Coding Club (Year 4)

One of the most memorable projects that I have made in my last year of coding club was Snake Game. The most enjoyable part of this project was bringing the classic Snake game to life using just Java. Watching it go from a static grid to a fully functional, interactive game—where the snake could move, eat apples, grow, and even die—was incredibly satisfying. It felt like real progress each time a new feature worked. The most difficult part was implementing the logic to check for self-collision and managing file I/O for saving and retrieving the high score. Parsing a text file to compare scores line by line took time to figure out, and debugging input validation with try-catch blocks was tricky at first. I chose this project because Snake is a simple yet timeless game that’s perfect for learning game logic, object movement, and condition handling. It was also a chance to challenge myself by building something more advanced than the usual console programs. I learned how important structure and planning are when building a game. From managing snake movement and direction, to drawing and refreshing the screen, to handling user input and saving scores—every part taught me how to write clean, logical, and efficient Java code. It also reminded me that breaking a big task into smaller parts makes even the hardest challenges manageable.
